92 """ 

93 Main entry point for Spark functionality. A SparkContext represents the 

94 connection to a Spark cluster, and can be used to create :class:`RDD` and 

95 broadcast variables on that cluster. 

96 

97 When you create a new SparkContext, at least the master and app name should 

98 be set, either through the named parameters here or through `conf`. 

99 

100 Parameters 

101 ---------- 

102 master : str, optional 

103 Cluster URL to connect to (e.g. mesos://host:port, spark://host:port, local[4]). 

104 appName : str, optional 

105 A name for your job, to display on the cluster web UI. 

106 sparkHome : str, optional 

107 Location where Spark is installed on cluster nodes. 

108 pyFiles : list, optional 

109 Collection of .zip or .py files to send to the cluster 

110 and add to PYTHONPATH. These can be paths on the local file 

111 system or HDFS, HTTP, HTTPS, or FTP URLs. 

112 environment : dict, optional 

113 A dictionary of environment variables to set on 

114 worker nodes. 

115 batchSize : int, optional, default 0 

116 The number of Python objects represented as a single 

117 Java object. Set 1 to disable batching, 0 to automatically choose 

118 the batch size based on object sizes, or -1 to use an unlimited 

119 batch size 

120 serializer : :class:`Serializer`, optional, default :class:`CPickleSerializer` 

121 The serializer for RDDs. 

122 conf : :class:`SparkConf`, optional 

123 An object setting Spark properties. 

124 gateway : class:`py4j.java_gateway.JavaGateway`, optional 

125 Use an existing gateway and JVM, otherwise a new JVM 

126 will be instantiated. This is only used internally. 

127 jsc : class:`py4j.java_gateway.JavaObject`, optional 

128 The JavaSparkContext instance. This is only used internally. 

129 profiler_cls : type, optional, default :class:`BasicProfiler` 

130 A class of custom Profiler used to do profiling 

131 udf_profiler_cls : type, optional, default :class:`UDFBasicProfiler` 

132 A class of custom Profiler used to do udf profiling 

133 

134 Notes 

135 ----- 

136 Only one :class:`SparkContext` should be active per JVM. You must `stop()` 

137 the active :class:`SparkContext` before creating a new one. 

138 

139 :class:`SparkContext` instance is not supported to share across multiple 

140 processes out of the box, and PySpark does not guarantee multi-processing execution. 

141 Use threads instead for concurrent processing purpose. 

142 

143 Examples 

144 -------- 

145 >>> from pyspark.context import SparkContext 

146 >>> sc = SparkContext('local', 'test') 

147 >>> sc2 = SparkContext('local', 'test2') # doctest: +IGNORE_EXCEPTION_DETAIL 

148 Traceback (most recent call last): 

149 ... 

150 ValueError: ...

748 def parallelize(self, c: Iterable[T], numSlices: Optional[int] = None) -> RDD[T]: 

749 """ 

750 Distribute a local Python collection to form an RDD. Using range 

751 is recommended if the input represents a range for performance. 

752 

753 .. versionadded:: 0.7.0 

754 

755 Parameters 

756 ---------- 

757 c : :class:`collections.abc.Iterable` 

758 iterable collection to distribute 

759 numSlices : int, optional 

760 the number of partitions of the new RDD 

761 

762 Returns 

763 ------- 

764 :class:`RDD` 

765 RDD representing distributed collection. 

766 

767 Examples 

768 -------- 

769 >>> sc.parallelize([0, 2, 3, 4, 6], 5).glom().collect() 

770 [[0], [2], [3], [4], [6]] 

771 >>> sc.parallelize(range(0, 6, 2), 5).glom().collect() 

772 [[], [0], [], [2], [4]] 

773 

774 Deal with a list of strings. 

775 

776 >>> strings = ["a", "b", "c"] 

777 >>> sc.parallelize(strings, 2).glom().collect() 

778 [['a'], ['b', 'c']] 

779 """ 

780 numSlices = int(numSlices) if numSlices is not None else self.defaultParallelism 

781 if isinstance(c, range): 

782 size = len(c) 

783 if size == 0: 

784 return self.parallelize([], numSlices) 

785 step = c[1] - c[0] if size > 1 else 1 # type: ignore[index] 

786 start0 = c[0] # type: ignore[index] 

787 

788 def getStart(split: int) -> int: 

789 assert numSlices is not None 

790 return start0 + int((split * size / numSlices)) * step 

791 

792 def f(split: int, iterator: Iterable[T]) -> Iterable: 

793 # it's an empty iterator here but we need this line for triggering the 

794 # logic of signal handling in FramedSerializer.load_stream, for instance, 

795 # SpecialLengths.END_OF_DATA_SECTION in _read_with_length. Since 

796 # FramedSerializer.load_stream produces a generator, the control should 

797 # at least be in that function once. Here we do it by explicitly converting 

798 # the empty iterator to a list, thus make sure worker reuse takes effect. 

799 # See more details in SPARK-26549. 

800 assert len(list(iterator)) == 0 

801 return range(getStart(split), getStart(split + 1), step) 

802 

803 return self.parallelize([], numSlices).mapPartitionsWithIndex(f) 

804 

805 # Make sure we distribute data evenly if it's smaller than self.batchSize 

806 if "__len__" not in dir(c): 

807 c = list(c) # Make it a list so we can compute its length 

808 batchSize = max( 

809 1, min(len(c) // numSlices, self._batchSize or 1024) # type: ignore[arg-type] 

810 ) 

811 serializer = BatchedSerializer(self._unbatched_serializer, batchSize) 

812 

813 def reader_func(temp_filename: str) -> JavaObject: 

814 assert self._jvm is not None 

815 return self._jvm.PythonRDD.readRDDFromFile(self._jsc, temp_filename, numSlices) 

816 

817 def createRDDServer() -> JavaObject: 

818 assert self._jvm is not None 

819 return self._jvm.PythonParallelizeServer(self._jsc.sc(), numSlices) 

820 

821 jrdd = self._serialize_to_jvm(c, serializer, reader_func, createRDDServer) 

822 return RDD(jrdd, self, serializer)

824 def _serialize_to_jvm( 

825 self, 

826 data: Iterable[T], 

827 serializer: Serializer, 

828 reader_func: Callable, 

829 server_func: Callable, 

830 ) -> JavaObject: 

831 """ 

832 Using Py4J to send a large dataset to the jvm is slow, so we use either a file 

833 or a socket if we have encryption enabled. 

834 

835 Examples 

836 -------- 

837 data 

838 object to be serialized 

839 serializer : class:`pyspark.serializers.Serializer` 

840 reader_func : function 

841 A function which takes a filename and reads in the data in the jvm and 

842 returns a JavaRDD. Only used when encryption is disabled. 

843 server_func : function 

844 A function which creates a SocketAuthServer in the JVM to 

845 accept the serialized data, for use when encryption is enabled. 

846 """ 

847 if self._encryption_enabled: 

848 # with encryption, we open a server in java and send the data directly 

849 server = server_func() 

850 (sock_file, _) = local_connect_and_auth(server.port(), server.secret()) 

851 chunked_out = ChunkedStream(sock_file, 8192) 

852 serializer.dump_stream(data, chunked_out) 

853 chunked_out.close() 

854 # this call will block until the server has read all the data and processed it (or 

855 # throws an exception) 

856 r = server.getResult() 

857 return r 

858 else: 

859 # without encryption, we serialize to a file, and we read the file in java and 

860 # parallelize from there. 

861 tempFile = NamedTemporaryFile(delete=False, dir=self._temp_dir) 

862 try: 

863 try: 

864 serializer.dump_stream(data, tempFile) 

865 finally: 

866 tempFile.close() 

867 return reader_func(tempFile.name) 

868 finally: 

869 # we eagerly reads the file so we can delete right after. 

870 os.unlink(tempFile.name)

2095 """

2097 different value or cleared. 

2098 

2099 Often, a unit of execution in an application consists of multiple Spark actions or jobs. 

2100 Application programmers can use this method to group all those jobs together and give a 

2101 group description. Once set, the Spark web UI will associate such jobs with this group. 

2102 

2103 The application can use :meth:`SparkContext.cancelJobGroup` to cancel all 

2104 running jobs in this group. 

2105 

2106 .. versionadded:: 1.0.0 

2107 

2108 Parameters 

2109 ---------- 

2110 groupId : str 

2111 The group ID to assign. 

2112 description : str 

2113 The description to set for the job group. 

2114 interruptOnCancel : bool, optional, default False 

2115 whether to interrupt jobs on job cancellation. 

2116 

2117 Notes 

2118 ----- 

2119 If interruptOnCancel is set to true for the job group, then job cancellation will result 

2120 in Thread.interrupt() being called on the job's executor threads. This is useful to help 

2121 ensure that the tasks are actually stopped in a timely manner, but is off by default due 

2122 to HDFS-1208, where HDFS may respond to Thread.interrupt() by marking nodes as dead.
