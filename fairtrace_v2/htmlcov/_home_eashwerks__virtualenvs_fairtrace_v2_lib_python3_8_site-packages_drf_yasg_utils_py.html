32def swagger_auto_schema(method=None, methods=None, auto_schema=unset, request_body=None, query_serializer=None, 

33 manual_parameters=None, operation_id=None, operation_description=None, operation_summary=None, 

34 security=None, deprecated=None, responses=None, field_inspectors=None, filter_inspectors=None, 

35 paginator_inspectors=None, tags=None, **extra_overrides): 

36 """Decorate a view method to customize the :class:`.Operation` object generated from it. 

37 

38 `method` and `methods` are mutually exclusive and must only be present when decorating a view method that accepts 

39 more than one HTTP request method. 

40 

41 The `auto_schema` and `operation_description` arguments take precendence over view- or method-level values. 

42 

43 :param str method: for multi-method views, the http method the options should apply to 

44 :param list[str] methods: for multi-method views, the http methods the options should apply to 

45 :param drf_yasg.inspectors.SwaggerAutoSchema auto_schema: custom class to use for generating the Operation object; 

46 this overrides both the class-level ``swagger_schema`` attribute and the ``DEFAULT_AUTO_SCHEMA_CLASS`` 

47 setting, and can be set to ``None`` to prevent this operation from being generated 

48 :param request_body: custom request body which will be used as the ``schema`` property of a 

49 :class:`.Parameter` with ``in: 'body'``. 

50 

51 A Schema or SchemaRef is not valid if this request consumes form-data, because ``form`` and ``body`` parameters 

52 are mutually exclusive in an :class:`.Operation`. If you need to set custom ``form`` parameters, you can use 

53 the `manual_parameters` argument. 

54 

55 If a ``Serializer`` class or instance is given, it will be automatically converted into a :class:`.Schema` 

56 used as a ``body`` :class:`.Parameter`, or into a list of ``form`` :class:`.Parameter`\\ s, as appropriate. 

57 :type request_body: drf_yasg.openapi.Schema or drf_yasg.openapi.SchemaRef or rest_framework.serializers.Serializer 

58 or type[no_body] 

59 

60 :param rest_framework.serializers.Serializer query_serializer: if you use a ``Serializer`` to parse query 

61 parameters, you can pass it here and have :class:`.Parameter` objects be generated automatically from it. 

62 

63 If any ``Field`` on the serializer cannot be represented as a ``query`` :class:`.Parameter` 

64 (e.g. nested Serializers, file fields, ...), the schema generation will fail with an error. 

65 

66 Schema generation will also fail if the name of any Field on the `query_serializer` conflicts with parameters 

67 generated by ``filter_backends`` or ``paginator``. 

68 

69 :param list[drf_yasg.openapi.Parameter] manual_parameters: a list of manual parameters to override the 

70 automatically generated ones 

71 

72 :class:`.Parameter`\\ s are identified by their (``name``, ``in``) combination, and any parameters given 

73 here will fully override automatically generated parameters if they collide. 

74 

75 It is an error to supply ``form`` parameters when the request does not consume form-data. 

76 

77 :param str operation_id: operation ID override; the operation ID must be unique accross the whole API 

78 :param str operation_description: operation description override 

79 :param str operation_summary: operation summary string 

80 :param list[dict] security: security requirements override; used to specify which authetication mechanism 

81 is requried to call this API; an empty list marks the endpoint as unauthenticated (i.e. removes all accepted 

82 authentication schemes), and ``None`` will inherit the top-level secuirty requirements 

83 :param bool deprecated: deprecation status for operation 

84 :param responses: a dict of documented manual responses 

85 keyed on response status code. If no success (``2xx``) response is given, one will automatically be 

86 generated from the request body and http method. If any ``2xx`` response is given the automatic response is 

87 suppressed. 

88 

89 * if a plain string is given as value, a :class:`.Response` with no body and that string as its description 

90 will be generated 

91 * if ``None`` is given as a value, the response is ignored; this is mainly useful for disabling default 

92 2xx responses, i.e. ``responses={200: None, 302: 'something'}`` 

93 * if a :class:`.Schema`, :class:`.SchemaRef` is given, a :class:`.Response` with the schema as its body and 

94 an empty description will be generated 

95 * a ``Serializer`` class or instance will be converted into a :class:`.Schema` and treated as above 

96 * a :class:`.Response` object will be used as-is; however if its ``schema`` attribute is a ``Serializer``, 

97 it will automatically be converted into a :class:`.Schema` 

98 :type responses: dict[str,(drf_yasg.openapi.Schema or drf_yasg.openapi.SchemaRef or drf_yasg.openapi.Response or 

99 str or rest_framework.serializers.Serializer)] 

100 

101 :param list[type[drf_yasg.inspectors.FieldInspector]] field_inspectors: extra serializer and field inspectors; these 

102 will be tried before :attr:`.ViewInspector.field_inspectors` on the :class:`.inspectors.SwaggerAutoSchema` 

103 :param list[type[drf_yasg.inspectors.FilterInspector]] filter_inspectors: extra filter inspectors; these will be 

104 tried before :attr:`.ViewInspector.filter_inspectors` on the :class:`.inspectors.SwaggerAutoSchema` 

105 :param list[type[drf_yasg.inspectors.PaginatorInspector]] paginator_inspectors: extra paginator inspectors; these 

106 will be tried before :attr:`.ViewInspector.paginator_inspectors` on the :class:`.inspectors.SwaggerAutoSchema` 

107 :param list[str] tags: tags override 

108 :param extra_overrides: extra values that will be saved into the ``overrides`` dict; these values will be available 

109 in the handling :class:`.inspectors.SwaggerAutoSchema` instance via ``self.overrides`` 

110 """

210def is_list_view(path, method, view): 

211 """Check if the given path/method appears to represent a list view (as opposed to a detail/instance view). 

212 

213 :param str path: view path 

214 :param str method: http method 

215 :param APIView view: target view 

216 :rtype: bool 

217 """ 

218 # for ViewSets, it could be the default 'list' action, or an @action(detail=False) 

219 action = getattr(view, 'action', '') 

220 method = getattr(view, action, None) or method 

221 detail = getattr(method, 'detail', None) 

222 suffix = getattr(view, 'suffix', None) 

223 if action in ('list', 'create') or detail is False or suffix == 'List': 

224 return True 

225 

226 if action in ('retrieve', 'update', 'partial_update', 'destroy') or detail is True or suffix == 'Instance': 

227 # a detail action is surely not a list route 

228 return False 

229 

230 if isinstance(view, ListModelMixin): 

231 return True 

232 

233 # for GenericAPIView, if it's a detail view it can't also be a list view 

234 if isinstance(view, (RetrieveModelMixin, UpdateModelMixin, DestroyModelMixin)): 

235 return False 

236 

237 # if the last component in the path is parameterized it's probably not a list view 

238 path_components = path.strip('/').split('/') 

239 if path_components and '{' in path_components[-1]: 

240 return False 

241 

242 # otherwise assume it's a list view 

243 return True

270def merge_params(parameters, overrides): 

271 """Merge `overrides` into `parameters`. This is the same as appending `overrides` to `parameters`, but any element 

272 of `parameters` whose ``(name, in_)`` tuple collides with an element in `overrides` is replaced by it. 

273 

274 Raises an ``AssertionError`` if either list contains duplicate parameters. 

275 

276 :param list[drf_yasg.openapi.Parameter] parameters: initial parameters 

277 :param list[drf_yasg.openapi.Parameter] overrides: overriding parameters 

278 :return: merged list 

279 :rtype: list[drf_yasg.openapi.Parameter]

282 parameters.update(param_list_to_odict(overrides))

287 """Remove ``None`` values from tuples, lists or dictionaries. Return other objects as-is. 

288 

289 :param obj: the object

291 """

444def field_value_to_representation(field, value): 

445 """Convert a python value related to a field (default, choices, etc.) into its OpenAPI-compatible representation.

448 :param object value: value 

449 :return: the converted value

463def get_field_default(field): 

464 """ 

465 Get the default value for a field, converted to a JSON-compatible value while properly handling callables. 

466 

467 :param field: field instance 

468 :return: default value 

469 """ 

470 default = getattr(field, 'default', serializers.empty) 

471 if default is not serializers.empty: 

472 if callable(default): 

473 try: 

474 if hasattr(default, 'set_context'): 

475 default.set_context(field) 

476 default = default() 

477 except Exception: # pragma: no cover 

478 logger.warning("default for %s is callable but it raised an exception when " 

479 "called; 'default' will not be set on schema", field, exc_info=True) 

480 default = serializers.empty 

481 

482 if default is not serializers.empty and default is not None: 

483 try: 

484 default = field_value_to_representation(field, default) 

485 except Exception: # pragma: no cover 

486 logger.warning("'default' on schema for %s will not be set because " 

487 "to_representation raised an exception", field, exc_info=True) 

488 default = serializers.empty 

489 

490 return default
