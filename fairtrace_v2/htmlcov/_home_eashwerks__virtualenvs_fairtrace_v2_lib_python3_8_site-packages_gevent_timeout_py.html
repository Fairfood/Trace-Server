3Timeouts. 

4 

5Many functions in :mod:`gevent` have a *timeout* argument that allows 

6limiting the time the function will block. When that is not available, 

7the :class:`Timeout` class and :func:`with_timeout` function in this 

8module add timeouts to arbitrary code. 

9 

10.. warning:: 

11 

12 Timeouts can only work when the greenlet switches to the hub. 

13 If a blocking function is called or an intense calculation is ongoing during 

14 which no switches occur, :class:`Timeout` is powerless.

72class Timeout(BaseException): 

73 """ 

74 Timeout(seconds=None, exception=None, ref=True, priority=-1) 

75 

76 Raise *exception* in the current greenlet after *seconds* 

77 have elapsed:: 

78 

79 timeout = Timeout(seconds, exception) 

80 timeout.start() 

81 try: 

82 ... # exception will be raised here, after *seconds* passed since start() call 

83 finally: 

84 timeout.close() 

85 

86 .. warning:: 

87 

88 You must **always** call `close` on a ``Timeout`` object you have created, 

89 whether or not the code that the timeout was protecting finishes 

90 executing before the timeout elapses (whether or not the 

91 ``Timeout`` exception is raised) This ``try/finally`` 

92 construct or a ``with`` statement is a good pattern. (If 

93 the timeout object will be started again, use `cancel` instead 

94 of `close`; this is rare. You must still `close` it when you are 

95 done.) 

96 

97 When *exception* is omitted or ``None``, the ``Timeout`` instance 

98 itself is raised:: 

99 

100 >>> import gevent 

101 >>> gevent.Timeout(0.1).start() 

102 >>> gevent.sleep(0.2) #doctest: +IGNORE_EXCEPTION_DETAIL 

103 Traceback (most recent call last): 

104 ... 

105 Timeout: 0.1 seconds 

106 

107 If the *seconds* argument is not given or is ``None`` (e.g., 

108 ``Timeout()``), then the timeout will never expire and never raise 

109 *exception*. This is convenient for creating functions which take 

110 an optional timeout parameter of their own. (Note that this is **not** 

111 the same thing as a *seconds* value of ``0``.) 

112 

113 :: 

114 

115 def function(args, timeout=None): 

116 "A function with an optional timeout." 

117 timer = Timeout(timeout) 

118 with timer: 

119 ... 

120 

121 .. caution:: 

122 

123 A *seconds* value less than ``0.0`` (e.g., ``-1``) is poorly defined. In the future, 

124 support for negative values is likely to do the same thing as a value 

125 of ``None`` or ``0`` 

126 

127 A *seconds* value of ``0`` requests that the event loop spin and poll for I/O; 

128 it will immediately expire as soon as control returns to the event loop. 

129 

130 .. rubric:: Use As A Context Manager 

131 

132 To simplify starting and canceling timeouts, the ``with`` 

133 statement can be used:: 

134 

135 with gevent.Timeout(seconds, exception) as timeout: 

136 pass # ... code block ... 

137 

138 This is equivalent to the try/finally block above with one 

139 additional feature: if *exception* is the literal ``False``, the 

140 timeout is still raised, but the context manager suppresses it, so 

141 the code outside the with-block won't see it. 

142 

143 This is handy for adding a timeout to the functions that don't 

144 support a *timeout* parameter themselves:: 

145 

146 data = None 

147 with gevent.Timeout(5, False): 

148 data = mysock.makefile().readline() 

149 if data is None: 

150 ... # 5 seconds passed without reading a line 

151 else: 

152 ... # a line was read within 5 seconds 

153 

154 .. caution:: 

155 

156 If ``readline()`` above catches and doesn't re-raise 

157 :exc:`BaseException` (for example, with a bare ``except:``), then 

158 your timeout will fail to function and control won't be returned 

159 to you when you expect. 

160 

161 .. rubric:: Catching Timeouts 

162 

163 When catching timeouts, keep in mind that the one you catch may 

164 not be the one you have set (a calling function may have set its 

165 own timeout); if you going to silence a timeout, always check that 

166 it's the instance you need:: 

167 

168 timeout = Timeout(1) 

169 timeout.start() 

170 try: 

171 ... 

172 except Timeout as t: 

173 if t is not timeout: 

174 raise # not my timeout 

175 finally: 

176 timeout.close() 

177 

178 

179 .. versionchanged:: 1.1b2 

180 

181 If *seconds* is not given or is ``None``, no longer allocate a 

182 native timer object that will never be started. 

183 

184 .. versionchanged:: 1.1 

185 

186 Add warning about negative *seconds* values. 

187 

188 .. versionchanged:: 1.3a1 

189 

190 Timeout objects now have a :meth:`close` 

191 method that *must* be called when the timeout will no longer be 

192 used to properly clean up native resources. 

193 The ``with`` statement does this automatically. 

194 

195 """

200 def __init__(self, seconds=None, exception=None, ref=True, priority=-1, 

201 _one_shot=False): 

202 BaseException.__init__(self) 

203 self.seconds = seconds 

204 self.exception = exception 

205 self._one_shot = _one_shot 

206 if seconds is None: 

207 # Avoid going through the timer codepath if no timeout is 

208 # desired; this avoids some CFFI interactions on PyPy that can lead to a 

209 # RuntimeError if this implementation is used during an `import` statement. See 

210 # https://bitbucket.org/pypy/pypy/issues/2089/crash-in-pypy-260-linux64-with-gevent-11b1 

211 # and https://github.com/gevent/gevent/issues/618. 

212 # Plus, in general, it should be more efficient

215 else: 

216 # XXX: A timer <= 0 could cause libuv to block the loop; we catch 

217 # that case in libuv/loop.py

220 def start(self): 

221 """Schedule the timeout.""" 

222 if self.pending: 

223 raise AssertionError('%r is already started; to restart it, cancel it first' % self) 

224 

225 if self.seconds is None: 

226 # "fake" timeout (never expires) 

227 return

229 if self.exception is None or self.exception is False or isinstance(self.exception, string_types): 

230 # timeout that raises self

232 else:

244 @classmethod 

245 def start_new(cls, timeout=None, exception=None, ref=True, _one_shot=False): 

246 """Create a started :class:`Timeout`. 

247 

248 This is a shortcut, the exact action depends on *timeout*'s type: 

249 

250 * If *timeout* is a :class:`Timeout`, then call its :meth:`start` method 

251 if it's not already begun. 

252 * Otherwise, create a new :class:`Timeout` instance, passing (*timeout*, *exception*) as 

253 arguments, then call its :meth:`start` method. 

254 

255 Returns the :class:`Timeout` instance. 

256 """ 

257 if isinstance(timeout, Timeout): 

258 if not timeout.pending: 

259 timeout.start() 

260 return timeout 

261 timeout = cls(timeout, exception, ref=ref, _one_shot=_one_shot) 

262 timeout.start() 

263 return timeout

265 @staticmethod 

266 def _start_new_or_dummy(timeout, exception=None, ref=True): 

267 # Internal use only in 1.1 

268 # Return an object with a 'cancel' method; if timeout is None, 

269 # this will be a shared instance object that does nothing. Otherwise, 

270 # return an actual Timeout. A 0 value is allowed and creates a real Timeout. 

271 

272 # Because negative values are hard to reason about, 

273 # and are often used as sentinels in Python APIs, in the future it's likely 

274 # that a negative timeout will also return the shared instance. 

275 # This saves the previously common idiom of 

276 # 'timer = Timeout.start_new(t) if t is not None else None' 

277 # followed by 'if timer is not None: timer.cancel()'. 

278 # That idiom was used to avoid any object allocations. 

279 

280 # A staticmethod is slightly faster under CPython, compared to a classmethod; 

281 # under PyPy in synthetic benchmarks it makes no difference. 

282 if timeout is None: 

283 return _FakeTimer 

284 return Timeout.start_new(timeout, exception, ref, _one_shot=True)

363def with_timeout(seconds, function, *args, **kwds): 

364 """Wrap a call to *function* with a timeout; if the called 

365 function fails to return before the timeout, cancel it and return a 

366 flag value, provided by *timeout_value* keyword argument. 

367 

368 If timeout expires but *timeout_value* is not provided, raise :class:`Timeout`. 

369 

370 Keyword argument *timeout_value* is not passed to *function*. 

371 """ 

372 timeout_value = kwds.pop("timeout_value", _NONE) 

373 timeout = Timeout.start_new(seconds, _one_shot=True) 

374 try: 

375 try: 

376 return function(*args, **kwds) 

377 except Timeout as ex: 

378 if ex is timeout and timeout_value is not _NONE: 

379 return timeout_value 

380 raise 

381 finally: 

382 timeout.cancel()
